      仅缓存的Bind的安装和使用

      2025-2-10
      Technology
      DNS

      安装配置 #

      在CentOS上安装：

      1
2
3
4

6
7

      $ yum install bind bind-utils -y
# 查看安装的版本
$ named -v
BIND 9.11.4-P2-RedHat-9.11.4-26.P2.el7_9.16 (Extended Support Version) <id:7107deb>
# 开机自启动
$ systemctl enable named
$ systemctl start named

      Bind安装完之后, 默认就是一个仅缓存的nameserver. 可以作为一个本地的DNS解析器来使用.
      Bind在我们的架构中也正是扮演这样的角色.
      在CentOS上安装完成之后, 默认的配置文件需要修改的不多:

      修改完成之后重启生效:

      1

      检查服务状态 #

      1. 先检查系统服务是否正常
      1

      systemctl status named

      如果看到active (running)说明服务运行正常.

      1. 检查服务日志

      可以通过检查named服务的日志来判断服务是否存在问题.

      1

      journalctl -u named -f

      重点关注error相关日志. 如果遇到network unreachable resolving 'registry-1.docker.io/AAAA/IN': 2a01:8840:9f::17#53类型日志, 往往是因为国内防火墙原因, 可以忽略.

      1. 检查监听的端口

      检查端口有没有正常监听, 也是判断服务是否运行正常的一个重要因素.

      1

      ss -nlp | grep 53

      观察udp和tcp两个协议的53端口是否监听, 以10.193.193.193这台服务器为例, 要保证下面的这些监听都要有.

      953端口是BIND服务的rndc(Remote Name Daemon Control)默认端口
